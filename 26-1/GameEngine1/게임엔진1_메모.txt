프로젝트 시작하면 맨 위에 asset, gameobject, component 메뉴가 있음
project, scene, asset, game object, component 생성 가능
object가 갖고 있는 기능은 component 통해서 구현 가능
project는 scene들의 집합, scene은 game object들의 집합, game object는 component들의 집합, asset은 component가 사용하는 데이터들의 집합(모델링, 오디오, 스크립트 등등)
텍스쳐 : 3d 모델링에 사용되는 이미지 데이터
유니티는 외부 프로그램들로 만들어진 여러 데이터들을 가져와서 연결시키기 위한 프로그램
asset들은 여러가지의 scene들에서 공유 가능

assets 메뉴에서 import new assets로 asset을 가져오거나, asset폴더 안에 파일을 복사해오면 프로젝트에서 사용 가능
모든 game object는 위치와 방향 component를 가짐
game object를 화면에 그리기 위해서는 mesh renderer와 mesh filter라는 component가 필요
또는 에셋을 드래그로 화면에 끌어올 수 있음
객체를 컨트롤 c, v, d, del 등 대부분 가능
가능하면 파일 정리, 삭제, 이동은 유니티 내의 project view창에서 할것
자료들을 트리 구조로 정리를 할 것

게임을 제작하다보면 무수히 많은 에셋들이 생기고, 필요한 자료를 찾는데 많은 시간을 할애하게 됨.
때문에 에셋의 이름을 잘 정의하여 검색하기 쉽게 만들거나, 에셋 유형으로 찾거나, 자주 사용하는 파일은 즐겨찾기로 미리 저장을 해놓는게 좋다
오브젝트를 만들때 트리구조로 만들어라
scene = 루트,  scene 밑에 많은 오브젝트들이 있고 그 밑에 오브젝트의 자식 오브젝트 등등이 있음
때로는 많은 자식 오브젝트를 묶어두기 위한 용도로 사용하는 빈 오브젝트도 있음, create empty object
부모 오브젝트의 영향은 자식 오브젝트도 받음
모든 오브젝트(처음 생성한 상태의 빈 오브젝트 포함)는 Transform component를 포함한다.
자식 오브젝트는 부모 오브젝트로부터 transform component의 정보를 상속받는다. 때문에 부모 오브젝트의 크기, 위치, 방향에 변화가 생기면 자식 오브젝트도 똑같이 바뀐다.
그러나 자식 오브젝트의 크기, 위치, 방향에 변화가 생겨도 부모 오브젝트에는 영향이 가지 않는다.
부모 오브젝트를 원점으로 자식 오브젝트의 위치를 표현 (오프셋)

게임오브젝트 여러개를 선택한다면 공통된 component를 가진 부분에 대해서 편집 가능

인스펙터 뷰의 항목들에서 값을 조절할 수있으면 벨류타입, 항목 옆에 동그란 버튼이 있으면 레퍼런스 타입, c#을 참고함, c#에는 포인터가 없고 레퍼런스가 그걸 대신함
