병렬전송 방식은 8개의 bit를 한번에 보냄, 직렬 전송 방식은 8개의 bit를 한줄로 순차적으로 보냄
과거에는 병렬이 더 빨랐으나 현재는 다름, 프린터 케이블 같은 곳에서 사용. 커넥터의 크기가 크기 때문에 기기의 소형화에 한계가 있음
usb(Universal Serial Bus) 저장장치에서 사용, 직렬전송 방식이 발전을 통해 현재는 병렬전송 방식보다 빨라짐

동기 전송은 수신자와 발신자가 양쪽 모두 준비가 되어야 데이터의 이동이 가능 ex) 전화 통화
비동기 전송은 발신자의 준비가 되지 않았어도 데이터의 이동이 가능 ex) 문자 메시지, 카톡 등등
중간 중간에 시작과 종료에 대한 데이터가 포함, 때문에 데이터의 양이 커질 수록 시작과 종료에 대한 의미없는 데이터가 커짐

simplex -> 한쪽 방향으로만 데이터가 이동 가능 ex) 과거 모니터, tv, 프린터기 
half-duplex -> 양쪽 다 데이터의 이동이 가능 하지만, 한 번에 한 쪽으로만 이동 가능 ex) 무전기
full-duplex -> 양방향 데이터의 이동이 가능, 과거에는 회선 설치 비용이 비싸서 양방향 데이터의 이동이 가능한 회선의 설치를 최소화 했지만, 현재는 사용이 늘어남.

데이터 표준 Data Communications Standards

표준 제정 단체에서 전문가들이 모여 초안을 발표함(RFC, Reguest For Comments) => 통상적으로 6개월동안 여러 피드백을 받아서 1차 표준안 제정 => 제정한 표준에 문제가 있을 경우 다시 RFC를 반복 => 공식 표준이 제정됨 Formal Standard
국가의 산업과 관련되어 있기 때문에 여러 전문가들을 제정에 참여시키려 함, 우리나라는 20년 밖에 안됨

LAN - Local Area Networks, 가장 작은 영역의 네트워크, 회사, 건물, 학교 단위, ex) 와이파이
MAN - Metropolitan Area Networks, LAN이 모여서 만들어진 네트워크, 대도시 단위
WAN - Waide Area Networks, MAN이 모여서 만들어진 네트워크, 국가 단위, ex) 인터넷, 스타링크
여기서 크기는 절대적이 아닌 상대적인 개념

LAN을 설계할때 사용자가 무엇을 필요로 하는 지 먼저 파악하고 거기에 맞춰 장비와 시스템을 구축
서버는 서비스를 제공하는 주체, ex) 메일, e-class, 통합정보시스템 등등
서버역할을 하는 컴퓨터가 필요, 네트워크 기능이 존재하는 운영체제가 탑재되어야 함
허브, 브릿지, 라우터 등의 다양한 장비들과 연결한다.
서버는 네트워크에 연결된 클라이언트에게 특정 서비스를 제공하는 주체
데이터 스토리지 서비스, 프린팅 서비스, 디렉토리 서비스 등등 다양한 서비스가 존재
과거에는 프린터 가격이 비싸서 한 대의 프린터를 프린트 서버로 많은 컴퓨터와 연결하여 여러 사람이 나눠서 사용, 그 이후에 프린트 가격이 낮아져 공급이 쉬워져서 잠시 사용을 안하다가, 회사 기밀 유출 같은 보안 문제가 생기자 유출을 막기 위해 프린트 서버를 다시 사용.

클라이언트(workstation)는 서버에게 서비스를 요청하고 서버에서 제공하는 서비스를 이용, 클라이언트 역시 네트워크 기능이 탑재된 운영체제가 탑재되어야 함

메인 프레임과 터미널은 과거 대형 컴퓨터를 사용하기 위해 사용, 더미 터미널은 계산을 못하고 입출력만 가능한 컴퓨터, 더미 터미널에서 입력된 데이터를 메인프레임으로 보내고, 메인 프레임이 계산해준 데이터를 받아서 출력함. 현재는 슈퍼 컴퓨터가 이와 비슷한 형식으로 사용

Peer-to-Peer(p2p) 서비스 - 성능이 비슷한 컴퓨터들끼리 연결된 서비스, 연결된 컴퓨터들이 업무를 나눠가짐, 컴퓨터가 서버가 되기도 하고 클라이언트가 되기도 함, 역할이 고정되어 있지 않음.
ex) 토렌트, 블록체인

client-dominant LANs 
클라이언트가 주가 되는 서비스, 서버는 클라이언트가 처리한 결과를 필요할 때 저장하는 역할, 개인용 pc의 성능이 높아지면서 생김

client/server LANs
서버와 클라이언트가 역할을 나눠서 사용, 주로 회사에서 직원들의 컴퓨터 사용내역을 감시하기 위해 사용

Distributed Processing LANs
특정 역할을 수행하는 서버를 나눠서 처리, ex) 게임 로그인 서버, 플레이 서버 등으로 나눠서 사용
서버를 유동적으로 사용하기 좋음

유행을 돌고 돈다. 과거에 사용했었지만 시간이 지나면서 점차 사용을 안하다가, 특정 상황, 장비의 발전 등으로 다시 사용하게 되는 경우가 있음. ex) 빅데이터 ai(챗지피티) - 사용자 <= 메인프레임 - 더미 터미널

thinnet을 이용해서 네트워크를 구성할때 선을 뽑아서 클라이언트에 연결시킬때 사용하는 커넥터
t형태
다양한 연결장치를 연결할때 이용 가능

Twisted Pair Media
쌍꼬임선, 두개의 얇은 구리선을 꼬와서 사용
최근에 LAN을 구성할 때 가장 보편적으로 사용되는 케이블
두개의 구리선을 꼬와 주변의 잡음에 강해짐
트위스티드 페어는 보호막이 있는지 없는지에 따라 STP(Shielded Twisted Pair)와 UTP(Unshielded Twisted Pair)로 구분된다.

UTP
초창기 UTP는 전화선으로 사용 (Cat(카테고리) 1 UTP)
점차 발전에 따라 현재는 Cat 6 UTP까지 개발
UTP 커넥터 - 8핀으로 구성, 별도로 RJ-45 Connectors로 많이 불려짐, 업계에서도 많이 불려지고 현재도 많이 사용중
유선 인터넷 연결에 자주 사용됨

STP
표피 안에 금속성 재질의 보호막으로 전자기적인 방해(EMI - ElectroMagnetic Interference)로부터 신호를 보호
UTP보다 신호에 잡음이 덜함
STP 커넥터 - 생김새는 UTP 커넥터와 비슷하지만, UTP 커넥터와 구분해주고, 안쪽의 금속성 보호막을 연결시켜줄 그라운드 쉴드가 필요하기 때문에 뒷면이 금속성 재질로 되어있음.
STP인지 UTP인지 구분하려면 커넥터의 뒷면을 확인하면 됨

Fiber-Optic Media
파이버옵틱 케이블, 광섬유 케이블
위의 두 케이블과 근본적인 차이가 존재, 전기 신호를 전송하기 위해 구리선을 사용하지만, 광섬유 케이블은 빛 신호를 전송하기 위해 유리나 빛이 통과할 수 있는 플라스틱 재질을 사용.
빛을 매개체로 하기 때문에 전송 속도가 매우 빠름
상당히 대용량의 데이터를 빠르게 이동시키기 위해 사용.
가격이 이전보다 싸져서 가정에서도 사용하는 사례가 생김
FTTH(Fiber To The Home) - 가정까지 광케이블로 네트워크를 구성했을때 그것들을 부르는 말
두개의 케이블이 하나의 세트, 한쪽은 보내는 용도, 나머지 하나는 받는 용도로 사용
구리선에 비해 광섬유는 내구성이 떨어지기 때문에 관리가 어려움, 손상이 생겼을 경우 데이터가 제대로 전송되지 않는 경우가 많음, 구부리지 않게 조심해야함
국가와 국가간의 연결을 위해 해저케이블 형태로 사용, 다만 쓰나미 또는 지진으로 인해 손상 위험성 있음
빛을 매개체로 사용하기 때문에 EMI로부터 자유로움

Female connectors, Male connectors가 있음
female 커넥터는 주로 장치에 달려있고 그것들을 선에 광섬유에 달려있는 male커넥터를 꽂아서 연결시켜줌

Wireless Media
무선 매체
유선매체에서 사용되는 커넥터가 필요없음
대신 주파수 대역을 사용, 라디오 주파수, Infrared, 마이크로웨이브 주파수 등이 있음

Radio Frequency
라디오 주파수
라디오 주파수를 이용하여 LAN을 구성할때는 transceiver(송신용)와 안테나(수신용)가 필요
통신용과 방송용 라디오 주파수는 구분되어있음.
용도에 맞게 주파수를 구분지어서 나눠주는 역할을 방송통신위원회 KCC에서 담당
가정용 무선 공유기에 주로 사용

Infrared
적외선 통신
라디오 주파수보다 통신거리가 짧기 때문에, 적은 양의 데이터를 근거리에서 보내는 용도로 사용
잡음에 취약
IRDA에 사용(적외선은 아니지만 블루투스와 비슷함)

Microwave
마이크로 웨이브 주파수
고주파이기 때문에 위의 무선매체들 보다 상대적으로 장거리 통신이 가능
통상적으로 건물과 건물 사이에 데이터를 이동 시킬때 사용
parabolic(파라볼릭) 안테나가 필요

무선매체에서는 별도의 특화된 커넥터가 필요 없음
대신에 안테나와 트랜시버가 커넥터의 역할을 대신함

Network Interface Card
네트워크 인터페이스 카드
네트워크 어뎁터 또는 네트워크 카드 혹은, NIC라고도 불림
전송매체와 통신을 담당하는 디바이스를 물리적으로 연결시켜주는 역할
케이블 포트가 탑재된 보드가 네트워크 인터페이스 카드이다.
현재는 보드형, usb형 등등 다양한 형태로 나온다.
NIC에 따라 최대 전송속도가 결정됨(통신사 서비스를 통해 아무리 빠른 속도의 인터넷을 제공받아도, 하드웨어가 못따라감)
어떤 architectures를 사용하냐에 따라 종류가 다름, 대부분은 이더넷 사용하지만 token ring, fddi, atm등의 다른 아키텍쳐를 사용한다면 nic도 그에 맞는 종류를 사용해야 함.
컴퓨터가 내보내는 데이터를 전송매체에 맞게 바꿔주거나 혹은 전송받은 데이터를 컴퓨터가 받을 수 있는 형태로 변환시켜줌
물리적인 node address(mac address) 제공
OSI 7계층 중에서 2계층(데이터링크)의 역할을 수행하고 있다.
수많은 업체들이 경쟁중이기 때문에 위에서 설명한 기능뿐만 아니라 다양한 부가기능을 탑재하는 경우가 많다. nic를 통해 컴퓨터의 전원을 끄고 키거나, nic를 통해 이동하는 데이터를 분석하는 관리 기능등을 제공하기도 한다. 이는 네트워크 관리자가 원격으로 장비를 관리하는데 도움을 준다.

Topology에 4종류가 있음
bus, star, ring, wireless 로 나뉨
토포로지는 조감도, 아키텍쳐는 건물 층계도 느낌

wireless LAN 아키텍쳐의 가장 많이 사례로 와이파이(IEEEE 802.11, 표준)가 있음
LAN보다 더 작은 규모의 네트워크를 PAN(personal Area Network)라고 하는데 블루투스가 해당됨

FDDI, ATM은 이더넷보다 성능이 좋지만 비싸고, 이더넷은 싸고 기술의 발전으로 성능이 좋기 때문에 주로 이더넷이 사용됨

토포로지는 로컬네트워크의 기본적인 구상도에 해당
bus, star, ring, wireless형으로 나뉨
상황에 맞는 형태를 사용해야 한다. ex) 노트북, 패드를 주로 사용하는 강의실이라면 wireless형을 이용
논리적 토포로지와 물리적 토포로지로도 나뉨

로지컬 토포로지는 데이터의 흐름을 나타낸다.
피지컬 토포로지는 네트워크 장비와 하드웨어, 케이블 등의 배치를 나타냄

비슷하다 = 같다
bus topology
버스 토포로지는 네트워크를 공유하는 형태로 구성됨.
네트워크상의 연결된 모든 디바이스가 데이터를 읽음.
이런 형태의 데이터 흐름을 broadcast라고 함(tv, 라디오) 
동축케이블을 주로 사용. 로지컬과 피지컬 토포로지가 일치
장점으로는 가격이 싸다, 동축케이블만 깔 수 있으면 된다.
단점으로는 모든 디바이스가 연결되어있으므로 케이블에 손상이 갈 경우 전체 네트워크에 문제가 생긴다.
또한 문제가 생겼을 경우 어디에 문제가 있는지 알기 어렵다.

star topology
스타 토포로지는 중간에 연결장치가 있고 연결장치와 디바이스를 UTP로 연결한 형태로 구성된다.
연결장치로는 허브나 스위치 등을 사용한다.
통상적으로 허브란 많은 사람들이 모이는 장소를 의미, 네트워크상의 허브는 데이터가 모이는 곳
허브는 데이터를 판단하거나 식별할 수 없어서 연결되어있는 모든 디바이스에 데이터를 보냄
허브를 사용한 스타토포로지는 로지컬에서는 버스형과 유사함
스위치는 데이터를 받으면, 누구한테 보낼 지 판단할 수 있어서 필요한 디바이스에게만 데이터를 보낸다.
스위치를 사용한 스타토포로지는 로지컬과 피지컬 둘다 스타토포로지다
unicast - 1 대 1 통신
장점으로는 LAN구축이 쉽다.
단점으로는 연결장치에 이상이 생겼을 경우 전체 네트워크에 문제가 생긴다.
케이블 하나에 손상이 생겼을 경우, 허브에서는 전체 네트워크가 다운이지만, 스위치에서는 일부만 손상된것과 같다.

ring topology
링 토포로지는 통상적으로 데이터가 한 방향으로만 돌아간다. (Unidirectional fashion)
실습실과 실습실 혹은 건물과 건물 사이의 데이터를 이동시키는데 사용 (우리 학교 건물 네트워크도 이런 형태라고 한다.) 또한 비싼 전송매체(광섬유)를 사용하는 경우가 많음
통상적으로 링을 2개 이상을 구축한다. 둘 중 하나는 비상 백업용으로 사용한다.(지진, 쓰나미 등등 대비)
혹은 우회로를 구축해놓는다.
단점으로는 케이블에 손상이 생기면 전체 네트워크가 다운되기 때문에 우회로 등을 구축해놓는다.

wireless topology
무선 토포로지는 주파수를 이용해서 데이터의 이동을 시킨다.
access points(엑세스 포인트)를 사용한다. (ex) 와이파이 공유기)
무선연결이 미치는 일정 범위를 CELL이라고 하는데 범위가 빈틈없도록 이루어진 네트워크를 셀룰로스라고 한다.
피지컬로는 스타와 비슷하다, 로지컬은 스타 혹은 버스와 비슷하다.
장점으로는 구축이 쉽고, 벽을 뚫을 필요도 없다.
또한 엑세스 포인트만 이동시키면 되기 때문에 쉽다.
단점으로는 엑세스 포인트에 가까이 있어야 하고, 보안에 취약하다.
또한 아직은 유선 네트워크보다 가격이 나간다.

엑세스 메소드
데이터를 어떤 규칙으로 전송매체에 실어보낼지 정의한 규칙

이더넷은 CSMA/CD 사용
Carrier Sense - 이더넷에 연결된 디바이스가 데이터를 전송하기 전에 이동중인 데이터가 있는지 감지하고 없을 경우 데이터를 보낸다
Multiple Access - 네트워크에 연결된 모든 디바이스가 동등한 엑세스 권리를 가진다 -> 선착순
Collision Detection - 데이터의 충돌을 감지하고, 두 디바이스가 랜덤한 시간동안 대기 후 다시 데이터를 전송시킨다. (두 디바이스가 우연히 동시에 데이터를 전송할 경우, 이동중인 데이터가 없다고 나왔는데 충돌날수가있음)

이더넷의 장점은 광범위하게 사용할 수 있다.
단점으로는 네트워크에 연결된 디바이스가 많으면 많을 수록 충돌 혹은 지연시간이 길어지므로 적당한 수의 디바이스 연결을 유지시켜야한다. 혹은 네트워크를 분리시켜 또다른 이더넷을 구축한다.
이때문에 가이드 라인이 존재한다.

이더넷은 기술 개발이 활발히 이루어지고, 이전 버전과 호환이 좋다.

디바이스들에게 순차적으로 보낼 데이터가 있는지 물어본다. 마스터 디바이스가 다른 디바이스들에게 물어봐야만 데이터를 주고받을 수 있기 때문에 다른 기기들이 보내는 데이터들과 충돌나지 않는다. 그러나 슬레이브들이 많아질수록 마스터가 물어봐야할 디바이스들이 많기 때문에 속도가 느려진다. 요즘은 마스터를 센트럴, 슬레이브를 퍼리프럴(peripheral, 주변)이라고 부른다. 과거 자료에는 여전히 마스터, 슬레이브로 나올 수 있음

Ch5 Network Operating System

NOS
서버 하드웨어와 네트워크 간 중재역할, 인터페이스 역할을 하는 소프트웨어다.
서버 컴퓨터에 탑재되어 구동된다.
네트워크를 타고 서버에 특정 서비스를 요청하는 클라이언트들의 접근에 대한 제어부분을 담당.
NOS는 클라이언트에도 탑재되어 구동되는데, 서버에 클라이언트가 원하는 특정 서비스를 요청하고 결과를 서버로부터 받아 클라이언트에게 넘겨주는 역할도 수행한다.

토렌트와 같은 p2p 서비스를 이용할 때 다른 곳에 있는 데이터를 받을때는 사용자의 디바이스가 클라이언트의 역할을 하고, 반대로 다른 디바이스로 데이터를 전송할때는 서버의 역할을 하는데, 이 일을 모두 할 수 있는 이유는 NOS가 탑재되어 있기 때문이다.

NOS가 제공하는 기능들
NOS가 탑재된 다양한 종류들의 서버들이 이와 같은 서비스를 제공 가능
- 파일 시스템 서비스
- 프린팅 서비스
- 어플리케이션 서비스 (게임 등)
- 웹 폴더 공유
- 웹 기반 관리
- 서버 클러스터링
- 디렉터리 서비스 - 데이터 베이스보다 가벼운 개념, 디렉터리는 저장된 데이터에 접근에 용이
- 고급 보안 시스템
- 인터넷, 웹 서비스

NOS의 발전 과정
초창기 NOS가 보급된 계기는 비싼 하드웨어 가격때문이다.
1980년대 초에는 비싼 하드 드라이브 가격때문에 하드 드라이브가 없는 컴퓨터를 사용, 대신 네트워크 서버에 연결하여 데이터를 저장하고 여러 사용자들과 공유.


하드디스크 가격이 내려가고 개인용 컴퓨터가 발전하면서 각자의 저장소에 자료를 저장하여 사용했으나, 기업에서는 데이터 보안 및 관리를 이유로 네트워크 시스템의 사용을 확대함.

클라우드 서버와 같은 저장소에 데이터를 저장하고, 단말기에 관계 없이, 언제 어디서나 데이터를 편리하게 접근하기 위해 사용하기도 함.

이동식 저장 매체의 발전에 따라 플로피 디스크, cd, 프린트 출력물 등의 매체로 기업의 데이터가 외부로 유출되는 일이 많아지자, cd룸 등의 부분을 막아버리거나, 프린팅 서버를 이용한 출력물 관리 이력 시스템을 도입하게 됨.

같은 네트워크 서비스지만 사용 목적은 과거와 많이 달라짐(과거 : 비싼 하드웨어의 가격, 현재 :보안 및 편리성)

Linux
UNIX에 뿌리를 두고있는 작은 버전의 UNIX
Linus Torvalds라는 필란드의 대학원생이 91년에 개발
Linux 1.0버전은 94년에 공개
오픈소스 기술을 적용했기에, 소스코드를 수정해서 기능을 추가하거나 바꾸는 것도 가능하다.
Novell NetWare나 윈도우즈가 제공하는 대부분의 서버 기능을 탑재
초기 인터넷 서버에 상당히 많이 사용
네트워크 기능이 탑재된 윈도우즈의 안정성이 매우 떨어졌고, 라이센스 비용을 내지 않고 무료로 사용이 가능했기 때문이다.
그러나 UNIX에 뿌리를 두고 있기 때문에 사용하기 어려웠다.
IBM, HP, Novell 등의 회사들이 지지한다. 이 회사들의 공통점은 마이크로 소프트와 대립하고 있다는 점이다.

UNIX
1960년대에 MIT와 GE, Bell 연구소들이 개발한 운영체제
주로 메인프레임 컴퓨터에서 사용하기 위한 운영체제
첫번째 버전은 1971년에 bell 연구소가 출시
70년대에 추가적인 버전들을 bell 연구소가 출시하여 무료로 공급
후에 메인프레임, 워크스테이션급 컴퓨터들을 생산하는 회사에서 유닉스를 운영체제로 사용하기 시작하면서 유닉스를 유료화함.
이때 유료화로 인해 UNIX의 전문가 양산에 차질이 생길 것을 우려해, AT&T Bell Labs에서 무료버전을 BSD UNIX라는 이름으로 버클리 캠퍼스에 제공
그 후 메인프레임을 생산하는 회사에서 본인들의 하드웨어에 맞춰 수정한 버전의 UNIX들을 출시했다.
현재는 UNIX의 비중이 줄긴 했지만 그럼에도 NOS의 대모격인 역할을 수행하고, 전문가들이 많이 없어서 유닉스를 잘 다룰수 있게 된다면 상당히 대우받을 가능성이 생긴다.
